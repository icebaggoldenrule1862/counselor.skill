# v1.0 成品目标

这份文档用来把仓库的成品版本钉死。

从现在开始，后续每一轮都不再泛泛扩展，而是围绕这个 `v1.0` 收口。

## 1. v1.0 一句话定义

`v1.0` 不是“功能最多的版本”，而是第一个真正像产品的版本：

> 一个能让大学生从零开始创建、蒸馏、验真、排练、现实同步，并把真实反馈带回来继续变准的单一辅导员系统。

## 2. v1.0 必须做到的事

### 1. 用户能从零开始

第一次来的人，就算没有整理好素材，也能：

- 先创建一个基础辅导员
- 跑一个短预演
- 知道下一步该补什么

### 2. 蒸馏流程足够顺

用户要能看懂：

- 先补什么素材最值钱
- 不同素材分别决定什么层的真实性
- 什么时候算“已经够真，可以开始拿去现实里练”

### 3. 真实性不是口号

仓库里必须有明确证据说明：

- 这套蒸馏为什么像
- 现在哪些模式已经能现实同步
- 还缺什么素材会导致哪里不够真

### 4. 现实同步链路成立

用户不能只是在这里“玩一轮”。
仓库必须清楚展示：

- 第一条消息怎么练
- 被追问后的下一句怎么练
- 说明文怎么起
- 求助沟通怎么开
- 用完以后怎么把真实反馈带回来

### 5. 同一个辅导员在不同用途下保持一致

这个项目最终卖点不是“题材多”，而是：

- 请假时像同一个人
- 求助时也像同一个人
- 办公室里还是像同一个人
- 写说明文时依然像同一个人

### 6. GitHub 首页一眼能懂

陌生访客应该很快看懂：

- 这是一个什么项目
- 怎么开始用
- 为什么蒸馏真实性是核心
- 为什么它不是请假模板库

## 3. v1.0 明确不做什么

`v1.0` 不做：

- 多角色联动
- 花哨玩法扩展
- 顶层事件菜单做成题材库
- 为了热闹而继续堆 prompt

## 4. v1.0 验收标准

如果 `v1.0` 做完了，至少应该满足：

1. 仓库里有清楚的成品目标和收口计划
2. README 已经像发布页，而不是内部笔记
3. 有一条完整的首次使用闭环说明
4. 有一组多用途现实同步样例
5. 有真实性报告和系统快照
6. 有真实反馈回流说明
7. 整个系统始终围绕单一辅导员一致性展开

## 5. 后续一切工作的判断标准

后面每次做新东西，都先问一句：

> 它是在让 v1.0 更接近可发布的成品，还是只是让仓库看起来更大？

只有前者值得继续做。
