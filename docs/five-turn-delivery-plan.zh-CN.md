# 五轮收口计划

用户要求尽量在接下来 5 轮内把项目推到高质量成品状态。
这份文档就是接下来默认执行的交付路线。

## Round 1：成品定义与首次使用闭环

目标：

- 钉死 `v1.0` 定义
- 设计从零创建到第一次现实同步的完整路径
- 补齐 onboarding 文档和样例

交付：

- `v1.0` 成品目标文档
- 五轮计划文档
- 首次使用闭环说明
- onboarding 样例组

## Round 2：真实反馈回流闭环

目标：

- 把“现实里用了之后怎么带回来”做成更完整的样例链路
- 让 correction、session、archive 的回流关系更清楚

交付：

- 连续使用示例
- 真实回复回流示例
- correction / session / archive 串联说明

## Round 3：跨模式一致性强化

目标：

- 让开口、追问、办公室、说明文、求助、复盘的“同一个人感”更强
- 做更明确的前后对比样例

交付：

- 跨模式一致性规则
- 一组对比样例
- 系统样例 README 强化

## Round 4：发布级展示强化

目标：

- 把 README、目录入口、关键样例整理成真正适合发 GitHub 的结构
- 让陌生访客更容易快速理解

交付：

- 发布级 README
- 核心示例索引
- 发布清单

## Round 5：v1.0 收口与验收

目标：

- 清理残余分散路线
- 做最终一致性检查
- 把仓库收成一个可发布的 `v1.0`

交付：

- 最终验收文档
- 剩余链路修正
- 发布前收口说明

## 默认执行原则

如果用户后面只说“继续”，就默认按这条路线往下推，不再每轮重新立项。
