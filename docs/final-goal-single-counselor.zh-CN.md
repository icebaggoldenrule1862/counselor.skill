# 最终目标：高质量单一辅导员系统

这份文档用来把项目的北极星重新钉死。

结论很明确：

> 这个项目最终不做多角色宇宙，不做群像式扩展，只做一个高质量、可长期使用、足够真实、能现实同步的单一辅导员系统。

## 1. 为什么要这样收口

这个项目最有价值的，不是角色数量。
真正重要的是下面这些体验能不能做深：

- 这个辅导员像不像
- 开口、追问、约谈、说明文、复盘之间是不是同一个人
- correction 之后整个系统是不是真的会跟着变
- 蒸馏之后是不是真的更像某个真实学校生态
- 连续几次使用后，它是不是已经开始“认识你”
- 练完以后，用户是不是真的敢把结果同步到现实里

如果这些还没做强，就继续扩玩法，只会把系统做散。

## 2. 最终产品一句话

这个项目最终应该是：

> 一个面向大学生真实沟通困境的高保真辅导员交互系统生成器。

它的任务不是让很多角色一起演戏，而是：

- 生成一个像你学校导员的人
- 让这个人在不同模式下保持一致
- 让这个系统会记忆、会纠偏、会蒸馏、会进化
- 让用户真的能先在这里练，再去现实里说

## 3. 最终版本最重要的 6 个能力

### 1. 单一人格极强一致性

同一个辅导员在这些模式下都要像同一个人：

- 开口器
- 追问应对器
- 办公室预演器
- 说明文工坊
- 补救器
- 复盘器

### 2. 深场景压力真实

不是靠喊人，而是靠：

- 证据节奏
- 态度判断
- 风险升级
- 收口方式

做出真实压迫感。

### 3. correction 真正改系统

用户说“不像”之后，不是只改一个输出，而是：

- 开口方式变
- 追问方式变
- 约谈节奏变
- 复盘重点变

### 4. 蒸馏真正进入学校生态

蒸馏不能只提几句口头禅。
它应该能把下面这些层次带进来：

- 通知语气
- 私聊追问方式
- 办公室施压节奏
- 说明文批注标准
- 制度升级路径

### 5. 长期记忆和档案闭环

系统要记住：

- 你最近在练什么
- 你最常卡在哪
- 哪些风险还在积累
- 导员现在怎么看你
- 下一步最该进入哪个模式

### 6. 现实同步价值清楚可见

陌生访客和真实用户都应该很快看明白：

- 这是一个系统，不是 prompt 拼盘
- 它蒸馏出来的人物够真
- 它能给出可直接拿去现实里用的结果
- 现实反馈回来以后，它会继续变准

## 4. 最终版本不做什么

明确不做：

- 多角色群像戏
- 顶层角色切换菜单
- 同时演很多人的复杂剧场
- 为了热闹而扩角色

这不是保守，而是战略收口。

## 5. 最终版本的成功标准

如果最终版本成熟了，用户应该会有这些感受：

- “这个导员真的像。”
- “我不是在玩一轮，我是在养一个系统。”
- “它知道我老卡在哪里。”
- “我一句‘不像’，它下轮真的就变了。”
- “我导入材料之后，它更像我们学校那个管理逻辑了。”
- “它给我的那条消息，我真的敢发。”

## 6. 后续推进原则

后面无论补什么，都要先问一句：

> 这件事是在让单一辅导员系统更强，还是在把项目做散？

只有前者，才值得继续做。
