# 最终版本推进线

这份文档不是路线图总览，而是接下来面向“成熟、耐玩、可长期用”版本的默认推进线。

如果后续用户只说一句“继续”，默认沿这 8 轮主线往前压。

## Round 1：长期档案层

目标：

- 让系统不只是记住 sessions 和 corrections
- 还要开始记住“这个学生长期处在什么状态”

本轮产物：

- 学生长期档案 schema
- archive 更新 prompt
- archive writer 工具
- 样例 archive

## Round 2：深场景状态机

目标：

- 把办公室约谈、考勤预警、宿舍违规这类高压场景做成更稳定的状态机

本轮重点：

- 状态字段扩展
- 证据链节奏
- 结果档位更细

## Round 3：档案影响场景

目标：

- 让长期档案真的反过来影响新场景

本轮重点：

- 档案中的印象、风险、历史主题进入新案例
- 同一用户新旧场景之间出现连续性

## Round 4：人格进化加强

目标：

- 让 correction 不只是修一轮，而是稳定影响多种模式

本轮重点：

- correction 优先级
- correction 前后跨模式对比
- correction 进入快照与档案

## Round 5：蒸馏增强加强

目标：

- 让通知、聊天、制度文本进入更真实的学校生态层

本轮重点：

- 冲突合并
- 学校制度语气提炼
- 蒸馏前后对比链路

## Round 6：单辅导员一致性强化

目标：

- 让单一辅导员在所有高价值模式里都像同一个人
- 不靠扩角色，也能把耐玩度和真实感继续往上推

本轮重点：

- 做跨模式一致性验证清单
- 做 correction / 蒸馏 / archive 三者共同影响下的系统一致性样例
- 收紧所有容易让人格串味的边界

## Round 7：学校差异模板

目标：

- 让不同学校、学院、管理风格开始真正分化

本轮重点：

- 学校气质参数
- 学院风格模板
- 场景与规则差异展示

## Round 8：发布强化与准 1.0 收口

目标：

- 把仓库变成真正适合公开展示、试玩和理解的版本

本轮重点：

- 首页收口
- 最佳样例顺序
- 发布清单
- 首版范围冻结

## 默认规则

如果中途发现某一层闭环明显更短板，允许穿插修补。

但大方向不变：

> 先让系统具备长期连续性，再让系统变复杂、变耐玩、变像真实学校生态。
