# 同一事件对比：逃课，在不同系统模式下会怎么完全不同

这份样例继续说明：

同一件事，不同模式下，系统应该像不同工具一样工作。

## 模式 1：约谈前预演

### 用户状态

- 明天要被辅导员谈逃课
- 很慌，不知道对方会怎么问

### 系统重点

- 先压迫感
- 再证据
- 再收口

### 样例

```text
辅导员：
你先别解释。
这门课老师已经把考勤发给我了，不是今天第一次。
```

## 模式 2：真实对话复盘

### 用户状态

- 已经谈完了
- 现在回想起来，不知道哪里说坏了

### 系统重点

- 拆关键转折点
- 指出哪句话最伤信任

### 样例

```text
你前面一直说“我不是故意的”，这句话的问题不在事实，而在于它没有回答辅导员真正关心的内容：
这是不是第一次，以及你为什么没有提前沟通。
```

## 模式 3：补救沟通

### 用户状态

- 约谈后还需要再发一条消息
- 想把事情拉回可处理范围

### 系统重点

- 补态度
- 补行动
- 不继续空泛解释

### 样例

```text
老师，今天这件事我回去之后重新想了一下，确实是我前面处理得太轻了。
我会先把情况说明补给您，也会把这门课后面的出勤和老师那边的要求认真跟上。
```

## 结论

同样是“逃课”：

- 约谈前预演，是高压模拟
- 真实对话复盘，是结构拆解
- 补救沟通，是局势修复

这说明事件只是壳，模式才是能力。
