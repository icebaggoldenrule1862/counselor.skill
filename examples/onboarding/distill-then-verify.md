# 蒸馏后再验真

这份样例回答：

> 用户已经补了一点素材之后，怎样判断“现在已经够真，可以开始拿去练现实问题”？

## 一个最小可用蒸馏包

推荐用户至少先补这三类：

1. 一段私聊 / 追问记录
2. 一条通知 / 公告
3. 一句用户手打的最像 / 最不像

如果再多一点，优先补：

4. 办公室或班会转写
5. 制度文本
6. 批注 / 审批反馈

## 验真不是只看像不像官腔

用户真正该看的，是：

1. 第一条消息像不像
2. 被追问后的接话像不像
3. 办公室节奏像不像
4. 说明文标准像不像

## 一个成熟的判断方式

如果真实性报告已经显示：

- `opening` 可用
- `follow_up` 可用
- `office_talk` 可用
- `statement_writing` 可用

那就说明这套系统已经不是只能“看看”，而是可以开始用于现实同步。

## 什么时候还不建议直接拿去用

如果出现这些情况，就不要急着同步现实：

- 只有公开通知，没有私聊素材
- 只有口头禅，没有规则路径
- 用户连续说“不像”，但核心行为逻辑还没修
- 办公室和说明文像两个不同的老师

## 这个样例想证明什么

蒸馏不是“素材越多越好”那么简单。
更关键的是：

> 用户要知道什么时候已经够真，什么时候还只是看起来有点像。
