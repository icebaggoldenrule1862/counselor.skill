# 真实反馈带回来之后怎么办

这份样例回答：

> 用户已经把系统给出的结果拿去现实里用了，回来以后怎样继续让系统变准？

## 1. 用户带回来的不该只是结果

不要只问：

- “成功了吗”

更该问的是：

- 辅导员真实怎么回的
- 她第一轮卡的是什么
- 她有没有提前亮老师反馈或规则压力
- 哪句话比预期更敏感

## 2. 回流时的三个入口

### correction

当用户说：

- “她不会这么说”
- “她真实没那么硬”
- “她其实更在意材料，不是态度”

这时优先进入 correction。

### session

当这一轮已经形成新经验，例如：

- 这位辅导员第一轮其实更爱追时间线
- 她会在第二轮才开始提任课老师反馈

这时应沉淀成 session。

### archive

当这个主题已经连续出现，或者已经影响长期印象时，就要进入 archive。

例如：

- 临时请假已经不是第一次
- 缺勤和请假开始连在一起
- 导员对用户的准备度形成了长期判断

## 3. 一个典型小链路

1. 用户练了一条请假消息
2. 现实里真的发了
3. 辅导员真实回“为什么现在才说，老师那边你自己沟通过没有”
4. 用户把这句带回来
5. 系统修正追问逻辑
6. 系统记录这位辅导员更在意“先沟通任课老师”这层
7. 下次再练类似问题时，输出就更贴近现实

## 这个样例想证明什么

这个项目最值钱的地方不是一次写得像。
而是：

> 用户每次把真实反馈带回来，系统都会更接近那个真实导员。
