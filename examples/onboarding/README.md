# onboarding 样例

这一组文件专门服务于第一次来仓库的用户。

如果你想按“从零开始 -> 蒸馏 -> 验真 -> 现实同步 -> 反馈回流”这条线理解项目，推荐按下面顺序看：

1. [zero-to-first-sync.md](./zero-to-first-sync.md)
2. [distill-then-verify.md](./distill-then-verify.md)
3. [feedback-return-loop.md](./feedback-return-loop.md)

这组样例的意义不是补更多题材，而是把单一辅导员系统最关键的首次使用体验收成一条顺路。
