# 真实反馈回流前后对比

这份对比专门解释：

> 一条真实辅导员回复带回来之后，系统到底具体变了什么？

## 对应反馈

真实回复是：

> 具体说，为什么现在才讲？任课老师那边你自己沟通过没有，别等老师找到我这里你再来补。

## correction 层的变化

回流前，系统虽然已经知道周老师会较早提老师反馈，但还没有把“任课老师前置沟通检查”放到第一轮核心位置。

回流后，新 correction 明确写死了三件事：

- 临时请假类 follow-up 第一轮就检查任课老师沟通
- `teacher-contact` 成为首轮焦点之一
- 遇到“老师已经找到我这里”的表达，要优先判定为被动补报风险

## session 层的变化

回流前，session 更多强调的是：

- 用户容易把时间线说乱

回流后，新 session 进一步明确：

- 用户不只会漏时间线，还会漏“前置沟通动作”
- 下次练类似问题时，要先准备“任课老师那边是否已沟通”的应答

## archive 层的变化

回流前，长期档案里主要风险还是：

- 考勤印象偏弱
- 临时沟通容易被解读为准备不足

回流后，新增的长期信号是：

- `任课老师前置沟通缺失会放大不信任`
- 当前活跃主题新增了 `临时请假的前置沟通`
- 推荐模式从 `debrief -> statement-writing` 变成更优先 `follow-up -> statement-writing -> debrief`

## 这份对比证明什么

真实反馈回流的价值，不是简单地说一句“这次预测不够准”。
它真正证明的是：

> 系统会把真实回复拆成可立即修正的行为逻辑、可复用的本轮经验、以及会影响后续判断的长期趋势。
