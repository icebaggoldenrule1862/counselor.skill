# 多源蒸馏前后对比

这份对比专门展示：

> 当系统不只读一份通知，而是同时读通知、私聊、制度、班会转写、手打印象、批注后，会发生什么？

## 蒸馏前

系统更像是：

- 有 archetype
- 有用户 correction
- 有少量通知 / 聊天 / 制度信号

能看出大方向，但还不够像一个真实学校生态里的导员。

### 蒸馏前的问题

- 容易只看到“官话硬不硬”
- 还分不清公开通知和私下追问的差异
- 还不够清楚说明文、审批反馈、办公室谈话分别在乎什么

## 蒸馏后

多源 bundle 进入后，系统开始出现 4 种更细的层次。

### 1. 公开文本和私下追问被拆开了

- 通知 / 公告：
  - 更硬
  - 更像公开行政语气
- 私聊 / 追问：
  - 更像 progressive-pressure
  - 先追时间线，再慢慢抬证据

### 2. 班会和办公室谈话的压力节奏更清楚了

从班会 / 办公室转写里可以看到：

- `listen-first`
- `teacher-feedback-early`
- `monitor-feedback-mid`
- `family-link-present`

这会直接影响：

- office-talk 的证据队列
- 哪一轮开始变硬

### 3. 说明文工坊不再只是“写一份说明”

从批注 / 审批反馈里能提炼出：

- 很在意时间线
- 材料不完整会要求重写
- “补材料”“重新提交”本身就是导员工作风格的一部分

这会直接影响：

- statement-writing
- follow-up requirements

### 4. 用户手打印象被保留成高优先级约束

用户说：

- “不会一上来把话说死”
- “不太会第一轮就提家长”

这些内容不会被通知文本粗暴覆盖，而会成为：

- correction 候选
- 稳定行为约束

## 结果

蒸馏后，系统不只是“更像某个人说话”。

而是更像：

- 某种公开管理语气
- 某种私下追问方式
- 某种办公室施压节奏
- 某种说明文审批标准

共同组成的“学校生态里的导员系统”。

## 这为什么能同步到现实

因为现实里的大学生最常需要同步出去的，通常就是这 4 类内容：

- 第一条怎么发
- 被追问时下一句怎么接
- 明天办公室里最危险的问题会怎么来
- 情况说明到底该怎么写

而现在这套多源蒸馏，已经能分别给这 4 类输出提供对应证据：

- 私聊 / 手打印象：支持第一句和下一句
- 办公室 / 班会转写：支持约谈预演
- 批注 / 制度文本：支持说明文工坊

## 这份对比证明什么

这说明蒸馏系统真正该做的不是：

- 只加更多来源

而是：

- 让不同来源分别进入不同层
- 再统一 merge 成更真实、更耐玩的系统
