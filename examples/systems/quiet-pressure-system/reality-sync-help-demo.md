# 蒸馏后如何现实同步：求助沟通示例

这份样例想证明：

> 这套系统不只是服务于“违规后补救”，也能服务于很多大学生更难开口的求助型沟通。

## 1. 用户当前困境

用户最近状态很差，已经影响到上课和生活。
但最难的不是“写出求助内容”，而是：

- 怕自己说得太重，像情绪宣泄
- 怕自己说得太轻，又显得不值得被接住
- 怕辅导员把这件事误读成借口或推责

## 2. 为什么这里也需要蒸馏

因为不同辅导员对“求助”两个字的反应差别很大。

有的人会先接情绪，有的人会先问影响范围，有的人会马上往制度和流程走。
如果系统没蒸出来这个差别，就只能给出泛化安慰句，现实里往往不敢直接发。

## 3. 蒸馏后的系统此时知道什么

这位辅导员系统已经被校准到：

- 她不是完全冷的类型
- 她能接住求助，但不喜欢纯情绪表达
- 她更容易接住“状态变化 + 具体影响 + 主动说明”的表达
- 她会希望用户不要等到事情积累到更严重才说

## 4. 一个更贴近现实同步的输出

### 第一条消息

老师，我想跟您提前说一下我最近的状态问题。不是单纯情绪波动，已经开始影响到这段时间的上课和日常安排了。我不是想拿这个当理由推事情，所以先主动跟您说明，也想听听您觉得我接下来该怎么处理会更稳一些。

### 如果对方回“具体一点”

老师，我最近主要是作息和状态都乱掉了，这几天已经影响到注意力和到课情况了。我现在先跟您说，是怕再拖下去会更难收拾，不是想等出了问题再解释。

### 为什么这更像蒸馏后的结果

因为它没有把求助写成情绪长文，也没有把自己说得很轻。
它更符合这位辅导员能接住的表达方式：

- 先说明这不是临时找借口
- 再说明已经有现实影响
- 最后把话留在“想更稳地处理”上

## 5. 现实同步后要回流什么

如果用户真的发出去了，最关键的回流信息是：

- 辅导员是先问状态，还是先问影响范围
- 她有没有主动提学校资源、请假、心理中心、班级安排
- 她对“我不是想推事”这种表述是接住了，还是觉得还不够具体

这些都会继续校准系统的求助沟通模式。

## 6. 这个示例想证明什么

这个项目不该被理解成一个“违规应对工具”。
它同样想覆盖很多大学生现实里更难开口的部分：

> 当用户不是要圆事，而是想认真求助时，系统也能给出像真实辅导员会接住的表达路径。
