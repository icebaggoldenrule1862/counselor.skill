# 核心排练模式

这份文件定义项目的高价值核心模式。

不是按“学校发生了什么”切，而是按“学生现在最需要什么沟通帮助”切。

## 模式 1：开口前排练

### 用户真实需求

- 不知道第一句怎么说
- 怕一开口就被质疑
- 想先找一个更稳的说法

### skill 应输出什么

- 第一条消息的 3 个版本
- 每个版本的风险点评
- 导员最可能的追问

### 代表事件

- 请假
- 实习冲突
- 比赛、面试、考公、考研安排
- 想求助但不敢讲

## 模式 2：被追问后的应对

### 用户真实需求

- 导员开始追问细节
- 已经发过消息，但后面不会接
- 想知道什么说法最容易翻车

### skill 应输出什么

- 当前这一轮的更稳回复
- 再下一轮的预判
- 不能说的雷区

### 代表事件

- 请假理由被追问
- 夜不归宿被问时间线
- 逃课被问是不是第一次

## 模式 3：约谈前预演

### 用户真实需求

- 明天要进办公室
- 想先感受压迫感
- 想过一遍最可能的问答

### skill 应输出什么

- 3 到 5 轮高压对话
- 对方手里可能有什么证据
- 哪一轮最容易出事

### 代表事件

- 逃课
- 夜不归宿
- 班群发言翻车
- 宿舍矛盾升级

## 模式 4：聊崩后的补救

### 用户真实需求

- 刚才态度不好
- 说错话了
- 想知道该不该补发消息

### skill 应输出什么

- 局势判断
- 一条补救消息
- 现在更适合解释、认错还是先停

### 代表事件

- 已读不回后再沟通
- 被问急了嘴硬
- 回得太冲

## 模式 5：说明文/检讨/申请写作

### 用户真实需求

- 需要一份像学生真实会写的说明
- 不想太模板
- 又怕太随便

### skill 应输出什么

- 可直接发的版本
- 更正式的版本
- 风险表达提醒

### 代表事件

- 请假说明
- 情况说明
- 检讨
- 补充材料

## 模式 6：求助型沟通

### 用户真实需求

- 并不是违规，而是需要帮助
- 但怕自己不会说、说不清、显得矫情

### skill 应输出什么

- 试探型开口
- 正式型开口
- 追问后的展开路径

### 代表事件

- 家里有事
- 心理状态差
- 经济困难
- 实习和课程冲突
- 想争取通融

## 模式 7：争取通融

### 用户真实需求

- 明明不完全合规
- 但想争取一次空间
- 想知道怎么说才不让人反感

### skill 应输出什么

- 较稳的争取话术
- 最适合强调的点
- 哪些表达会让导员觉得你在讨价还价

### 代表事件

- 临时请假
- 晚交材料
- 错过流程后的补报
- 希望先不升级处理

## 模式 8：真实对话复盘

### 用户真实需求

- 已经发生过真实对话
- 想知道自己哪里错了
- 想知道下一次还能怎么补

### skill 应输出什么

- 关键转折点
- 导员视角下的敏感点
- 下一步沟通建议

### 代表事件

- 约谈结束后的复盘
- 被拒绝后的复盘
- 发消息翻车后的复盘
