# 从这里开始

这份文件专门回答一个问题：

> 用户第一次打开这个 skill，到底应该怎么开始？

核心原则：

- 不从“事件名”开始
- 从“用户现在卡在哪一步”开始

## 最推荐的起手问题

先问用户一句：

> 你现在最接近哪种状态？

然后按下面 6 类路由：

## 1. 还没开口，不知道第一句怎么说

典型情况：

- 想请假，但不知道怎么开口
- 想找导员说实习、比赛、缓考、回家、家里有事
- 怕一说就显得不合规、不成熟、不真诚

这个 skill 应该提供：

- 第一条消息的 3 个版本
- 哪种版本更稳
- 导员最可能怎么追问

推荐起手 prompt：

- “我明天想找辅导员请假，但这个理由不太好说，你先帮我排练第一句。”
- “我想跟导员说我最近状态不太行，但不知道怎么开口。”

## 2. 已经发出去了，但不知道下一句怎么接

典型情况：

- 导员回复很冷
- 只回了“？”、“什么事”、“具体说”
- 对方开始追问细节

这个 skill 应该提供：

- 接下来最稳的回复
- 哪种说法会让导员更起疑
- 如果对方继续追问，第二轮怎么接

推荐起手 prompt：

- “我已经给导员发消息了，他回我‘具体说’，后面我怎么接比较稳？”
- “辅导员说‘你先说清楚’，我现在不知道怎么回。”

## 3. 明天要去办公室，想先预演

典型情况：

- 明天要被约谈
- 知道对方手里可能有证据
- 想先过一遍最可能被问什么

这个 skill 应该提供：

- 高压多轮预演
- 最可能出现的追问
- 哪一轮最容易翻车

推荐起手 prompt：

- “明天辅导员要找我谈逃课，你先按最真实的方式跟我过一遍。”
- “我可能要被问夜不归宿，帮我模拟一下最可能的办公室对话。”

## 4. 已经聊崩了，想补救

典型情况：

- 刚刚态度不太好
- 说错话了
- 觉得导员已经不爽了

这个 skill 应该提供：

- 当前局势评估
- 一条补救消息
- 该不该继续解释
- 是先认错还是先补信息

推荐起手 prompt：

- “我刚刚回辅导员回得有点冲，怕事情升级，帮我想一条补救消息。”
- “我感觉刚才那段话说得不对，帮我补救一下。”

## 5. 需要写东西，但不想太模板

典型情况：

- 请假说明
- 情况说明
- 检讨
- 补充材料

这个 skill 应该提供：

- 一个像学生真实会写的版本
- 一个更稳妥的版本
- 哪些表达不能碰

推荐起手 prompt：

- “我要写一份情况说明，别写得太假太模板，你帮我起草。”
- “我要补一个请假说明，帮我写得像真的学生会发给辅导员的。”

## 6. 其实不是违规，是求助，但怕不会说

典型情况：

- 想请假回家
- 想说家里问题、心理状态、经济困难、实习冲突
- 不是为了骗，而是不知道怎么表达才不会显得麻烦

这个 skill 应该提供：

- 试探性开口版本
- 稍微正式一点的版本
- 如果导员继续追问，该怎么说

推荐起手 prompt：

- “我不是想骗假，我是真的最近状态不好，但我不知道怎么跟辅导员说。”
- “我想找导员商量实习和上课冲突，帮我排练一下怎么说。”

## 一个最重要的产品判断

这个 skill 的顶层单位不应该是：

- 请假
- 逃课
- 夜不归宿

而应该是：

- 开口前排练
- 被追问后的应对
- 明天约谈前预演
- 聊崩后的补救
- 说明文起草
- 求助型沟通

事件只是底层素材。
真正的产品入口是“沟通节点”。
