# 蒸馏真实性协议

这份文档回答的是：

> 一个被蒸馏出来的辅导员系统，怎样才算“真”，而不是只会模仿官腔？

这个项目的价值，不是让用户围观一个“像辅导员的角色”说话。
真正值钱的是：

- 用户敢把它当成现实排练场
- 系统给出的内容能更顺滑地同步到真实沟通
- 用户把真实反馈带回来后，系统还能继续修正

所以真实性不是装饰项，而是主干质量线。

## 1. “真”至少要过四层

一个成熟的蒸馏结果，至少要同时过这四层：

1. 语气真
公开通知、私聊、办公室谈话、说明文反馈，语气层级要分得开。

2. 追问逻辑真
这个辅导员先问什么，最敏感什么，会不会先听解释，再压时间线或材料，这些要稳定。

3. 规则路径真
事情会不会进入说明文、约谈、补材料、学院上报，这些升级路径要像真实学校生态。

4. 收口方式真
同样一句“你先写说明”，有的人是在给台阶，有的人是在正式留痕。收口方式必须像同一个人。

一句话说，真实性不只是“像他怎么说话”，还要“像他怎么处理事”。

## 2. 真实性优先级

蒸馏时不要先追口头禅。
更重要的优先级是：

1. 追问方式
- 第一轮会先问什么
- 更看重时间线、态度、影响、流程还是补材料
- 会不会先给你说话机会

2. 证据节奏
- 会不会第一轮就亮底牌
- 是先拿任课老师反馈压，还是先拿制度压
- 家长、学院、记录、处分这些高压点什么时候出现

3. 规则路径
- 什么问题会进入书面说明
- 什么问题会进入正式约谈
- 什么问题只是重点观察

4. 表层语气
- 常用句
- 常见开场
- 结尾口吻
- 官话浓度

表层语气重要，但不该放在第一位。

## 3. 什么素材决定什么真实性

不同素材解决的是不同层的“真”：

- 私聊 / 追问记录
决定第一句话、下一句、追问顺序、是否先听解释。

- 班群通知 / 学院公告
决定公开场景的语气硬度、纪律语言和留痕意识。

- 制度文本 / 请假规范 / 考勤办法
决定规则边界、升级路径和“这件事最坏会走到哪”。

- 班会录音转写 / 办公室约谈转写
决定高压场景的节奏感、证据揭示顺序和压迫方式。

- 说明文批注 / 审批反馈
决定说明文写作时会被挑哪些问题，哪些地方容易被打回。

- 用户手打印象
决定最不能跑偏的高优先级约束，比如“嘴上不凶，但会慢慢收口”。

## 4. 推荐蒸馏顺序

推荐用户按这个顺序蒸：

1. 先手打印象
先把“这个人最像什么、最不像什么”钉住。

2. 再补私聊或追问记录
先把接话逻辑校准出来。

3. 再补通知和制度
让公开语气和规则路径稳定。

4. 最后补办公室转写和批注
把高压场景和说明文标准做实。

这样蒸出来的系统更稳，也更容易直接拿去现实里用。

## 5. 真实性红线

出现这些情况，应该视为蒸馏还不够真：

- 开口、追问、约谈、复盘像四个不同的人
- 私聊明明会先听解释，系统却每次一上来就压死
- 明明规则压力很重，系统却只会写漂亮话
- 用户多次说“不像”，系统还是只改措辞，不改行为逻辑
- 说明文工坊和办公室预演的判断标准互相打架

## 6. 真实性检查清单

每次蒸馏完成后，至少检查：

1. 第一条消息像不像这个人会接得住的话
2. 被追问时，系统给的下一句稳不稳
3. 办公室预演时，证据是不是按真实节奏揭示
4. 写说明文时，系统会不会挑到真实会被卡的点
5. 同一套逻辑能不能同时贯穿开口、追问、约谈、复盘

## 7. 与现实同步的关系

真实性不是为了“更好看”，而是为了这条链路成立：

1. 用户蒸馏自己的辅导员
2. 在系统里先排练
3. 拿到一条能发、一句能接、一份能改的结果
4. 去现实里使用
5. 把真实反馈再带回来修正

如果蒸馏结果不真，这条链路会在第三步之前就断掉。

## 8. 最终目标

成熟版本里的蒸馏应该做到：

> 用户会觉得“这真的像我导员，所以我敢先在这里练一遍，再去现实里说”。
