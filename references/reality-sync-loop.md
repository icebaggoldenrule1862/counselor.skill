# 现实同步闭环

这份文档定义的是：

> 用户在系统里练完之后，怎样把结果更自然地同步到现实沟通？

这层很关键。

因为项目的价值不只是“模拟得好”，而是：

- 模拟之后，现实里真的更好用

## 1. 最理想的使用路径

推荐路径是：

1. 蒸馏导员
2. 选择当前困境
3. 拿到一条可直接用的消息 / 一段下一句 / 一份说明文结构
4. 去现实里使用
5. 把现实反馈带回系统
6. 更新 correction、session、archive

这个闭环一旦跑起来，系统就会越来越像，也越来越实用。

## 2. 现实同步的 4 种高价值输出

### 1. 可直接发出的第一条消息

适用：

- 请假
- 求助
- 说明情况

要求：

- 不像模板
- 不像论文
- 不要太像 AI 写的

### 2. 下一轮怎么接

适用：

- 对方回了“具体说”
- 对方回了“为什么现在才说”
- 对方开始质疑态度

要求：

- 给用户当前这一轮最稳的说法
- 顺带告诉他再下一轮最可能被问什么

### 3. 说明文结构

适用：

- 写请假说明
- 写情况说明
- 写补充材料

要求：

- 先给结构，再给草稿
- 明确哪些地方最容易被导员挑

### 4. 现实对话复盘

适用：

- 用户已经和导员聊过了
- 想知道哪里翻车

要求：

- 复盘要具体到句子
- 要指出下一次同步到现实时应该怎样改

## 3. 不要把现实同步做成什么

不要做成：

- 一大段长 prompt 教学
- 一堆抽象话术原则
- 花里胡哨但发不出去的文案

真正该做的是：

- 给一句能发的
- 给一轮能接的
- 给一份能改的

## 4. 现实同步成功的判断标准

一次同步如果成功，用户会有这种感觉：

- “这句话我真能发。”
- “这比我自己乱说稳多了。”
- “这份说明拿去改一下就能用。”
- “现实导员的反应没有和系统差太远。”

## 5. 把现实反馈带回来

现实同步后，最重要的不是结束，而是回流：

- 导员真实怎么回的
- 哪句话比预想中更敏感
- 哪个地方其实没有那么硬

这些都应该带回系统，进入：

- correction
- session
- archive

## 最终目标

这个闭环真正想做到的是：

> 系统不是现实沟通的替代品，而是现实沟通前后的练习场和校准器。
