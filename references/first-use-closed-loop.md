# 首次使用闭环

这份文档回答的是：

> 一个第一次来到仓库的用户，怎样才能在最短路径里真正用起来，而不是卡在概念和配置里？

## 1. 首次使用的核心目标

第一次体验最重要的不是“理解全部系统”，而是三件事：

1. 先捏出一个足够像的基础导员
2. 先拿到一轮现实可用结果
3. 先建立“这套系统会越来越准”的信心

## 2. 推荐的首次路径

### 第一步：先描述，不先搜集

让用户先说：

- 这个辅导员大概是什么风格
- 最像什么
- 最不像什么

不要一上来要求整理大量素材。

### 第二步：先预演，不先深蒸

先让系统给出：

- 一条开口
- 一轮接话
- 或一个短预演

用户先感受这个人像不像。

### 第三步：再补素材蒸馏

如果用户觉得有点像，但还不够准，再补：

- 私聊 / 追问记录
- 用户手打印象
- 通知 / 公告
- 班会或办公室转写
- 制度文本
- 批注 / 审批反馈

### 第四步：看真实性报告

真实性报告要帮助用户判断：

- 现在哪些模式已经足够稳
- 哪些模式还不建议直接拿去现实里用
- 下一步最值钱的素材是什么

### 第五步：开始现实同步

用户先从最小动作开始：

- 发一条消息
- 接一句追问
- 起一个说明文结构

### 第六步：把真实反馈带回来

真实用了以后，再带回：

- 对方真实怎么回的
- 哪一轮最危险
- 哪个地方和系统预演有偏差

这些会继续进入：

- correction
- session
- archive

## 3. 首次使用时最不该做的事

不要：

- 一上来先讲一堆抽象理念
- 一上来先让用户选很多功能
- 一上来先让用户上传所有材料
- 第一轮就把用户拖进复杂深场景

## 4. 首次使用成功的标志

如果第一次体验是成功的，用户通常会有这些感受：

- “我已经能看出这个导员像不像了。”
- “我已经拿到一条我真敢发的消息了。”
- “我知道下一步该补什么材料了。”
- “我知道这不是一次性聊天，而是一套会变准的系统。”
