# 场景库

这份文件定义首批高频校园事件，用于 scene engine 调用。

注意：

- 这里是底层事件素材库
- 不是用户第一次进入 skill 的顶层入口
- 顶层入口应优先来自 `start-here.md` 和 `rehearsal-playbooks.md`

## 使用要求

- 每个场景都要带“已知信息”和“隐藏证据”
- 不要把所有证据一开始全抛出来
- 每个场景都应有至少 3 条分支走向

## 场景 1：请假申请

### 触发语境

- 学生临时请假
- 请假理由模糊
- 辅导员怀疑真实性

### 已知信息

- 学生主动来请假
- 时间节点较近

### 隐藏证据

- 任课老师是否已经说明有测验
- 该学生近期请假频次
- 班长是否反馈过类似情况

### 结果方向

- 正常批准
- 补材料后批准
- 口头警告
- 转为重点关注

## 场景 2：逃课被抓

### 触发语境

- 任课老师点名缺席
- 学生被叫去办公室解释

### 已知信息

- 学生已经旷课一次或多次

### 隐藏证据

- 老师是否发过考勤截图
- 同学是否替答到失败
- 该课程是否已接近预警线

### 结果方向

- 写情况说明
- 口头警告
- 家长联动风险
- 学业预警升级

## 场景 3：上课玩手机或睡觉被反馈

### 触发语境

- 任课老师点名批评
- 辅导员收到反馈后二次约谈

### 已知信息

- 不一定缺课，但课堂表现差

### 隐藏证据

- 任课老师是否已有持续观察
- 是否影响班级形象
- 学生是否曾有相同问题

### 结果方向

- 提醒后结束
- 记一次重点谈话
- 列入课堂纪律关注名单

## 场景 4：夜不归宿

### 触发语境

- 宿管登记未归
- 查寝人数不齐

### 已知信息

- 学生未在规定时间返回

### 隐藏证据

- 宿舍长是否知情
- 是否存在多次未归
- 是否已触发学院红线

### 结果方向

- 写检讨
- 正式约谈
- 家长联系
- 处分风险

## 场景 5：宿舍矛盾升级

### 触发语境

- 室友投诉
- 班委反馈
- 群聊冲突外溢

### 已知信息

- 双方说法不一致

### 隐藏证据

- 其他室友站队情况
- 过去是否已有矛盾积累
- 是否涉及卫生、作息或财物问题

### 结果方向

- 调解成功
- 双方写说明
- 调宿风险
- 长期重点关注

## 场景 6：评优资格摇摆

### 触发语境

- 学生想争取奖学金、评优或保研材料
- 但近期表现有瑕疵

### 已知信息

- 学生带有明显目的性

### 隐藏证据

- 班委评价
- 纪律记录
- 老师私下印象

### 结果方向

- 资格保留
- 资格待定
- 资格受限
- 需额外补救表现

## 场景 7：班群发言翻车

### 触发语境

- 在班群、课程群或学院群中说错话
- 被截图转给辅导员

### 已知信息

- 发言已经被看见

### 隐藏证据

- 是否已引发更大范围传播
- 是否带有顶撞、阴阳或不当表述
- 是否被老师单独关注

### 结果方向

- 口头提醒
- 删除并道歉
- 重点谈话
- 公开处理风险

## 场景 8：实习与学院安排冲突

### 触发语境

- 学生想请假参加面试、实习或考试
- 与课程、会议、返校安排冲突

### 已知信息

- 学生有自己的现实规划压力

### 隐藏证据

- 学院对去向数据的要求
- 导员是否重视就业率
- 是否已多次请假

### 结果方向

- 协调通过
- 有条件批准
- 记一次纪律问题
- 贴上“不配合管理”标签
