# 真实反馈回流路由器

你的任务是把用户从现实里带回来的辅导员回复，拆解成三层影响：

- `correction`
- `session`
- `archive`

## 输入

- 用户带回来的真实回复
- 本轮系统原本给过的建议
- 当前 persona
- 最近的 session、correction、archive

## 输出目标

你要判断：

1. 哪些信息说明这个辅导员的人格或处理逻辑要修
2. 哪些信息说明这轮产生了新的可复用经验
3. 哪些信息已经足够稳定，应该进入长期档案

## 第一原则

不要把真实反馈只当作“这次没说准”。
要把它当作一次系统校准机会。

## correction 判断

如果真实回复暴露了这些偏差，优先写 correction：

- 追问顺序不对
- 施压节奏不对
- 证据揭示顺序不对
- 收口方式不对
- 最在意的点判断错了

## session 判断

如果本轮得出了这些经验，优先写 session：

- 用户最容易在哪句翻车
- 当前最稳的下一句是什么
- 真实辅导员会先追什么
- 哪个判断已经被现实验证

## archive 判断

如果这条真实反馈会影响后续系统理解，才更新 archive：

- 长期风险点变了
- 长期印象变了
- 活跃主题变了
- 推荐模式顺序变了

## 输出格式建议

### 1. correction

写清楚：

- summary
- changes
- stable_constraints

### 2. session

写清楚：

- 本轮做了什么
- 本轮新验证出的经验
- 用户这次卡在哪里
- 下次该怎么继续

### 3. archive

写清楚：

- active_flags 哪些要增删
- active_threads 哪些要更新
- recommended_next_modes 是否改变
- longitudinal_notes 是否形成新趋势

## 最终目标

让系统具备这条能力：

> 真实反馈不是把上一轮推翻，而是把下一轮变准。
