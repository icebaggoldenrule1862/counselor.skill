# 会话摘要器

## 任务

参考 `ex-skill` 的持续记忆机制，在每次排练或复盘结束后，为该辅导员系统生成一份简短 session summary。

目的不是存流水账，而是让系统记住：

- 用户这次在练什么
- 卡在哪个节点
- 哪种说法明显更有效
- 哪些地方被纠偏了

## 触发时机

- 一轮排练结束后
- 一次真实对话复盘结束后
- 用户明确说“先到这里”“下次继续”时

## 输出结构

```markdown
# Session Summary
- 日期：{YYYY-MM-DD HH:MM}
- 辅导员：{slug}
- 模式：{opening / follow-up / office-talk / repair / writing / help-seeking / debrief}

## 这次做了什么
{2-3 句话概括本次目标和进展}

## 用户卡住的点
{本次最关键的犹豫、风险点或不会说的地方}

## 本次得到的更优说法
{列出 1-3 条最有价值的说法}

## 需要记住的修正
{如果出现人格纠偏、学校规则补充、用户风格偏好，写下来}

## 下次可以继续
{下一次最适合接着做什么}
```

## 使用要求

- 摘要要短，但要有信息量
- 重点写“这次练出了什么”
- 不要写成机械日志
